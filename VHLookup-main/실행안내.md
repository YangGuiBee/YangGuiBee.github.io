# VHLookupLocal 실행 안내

비개발자 사용자는 Windows 실행파일을 내려받아 더블클릭하면 됩니다.

현재 버전: `V1.1`

- [VHLookup Local V1.1 Windows 실행파일 다운로드](https://github.com/koul777/VHLookup/releases/latest/download/VHLookupLocal_pivot_v1.1.exe)
- 직접 다운로드가 안 되면 [GitHub Releases](https://github.com/koul777/VHLookup/releases/latest)에서 `VHLookupLocal_pivot_v1.1.exe`를 내려받습니다.

실행파일은 소스 저장소의 파일 목록에 포함하지 않습니다. exe 파일은 GitHub Release 첨부파일로 배포합니다.

소스 ZIP을 받거나 저장소를 clone한 경우 `dist\VHLookupLocal_pivot_v1.1.exe`는 없는 것이 정상입니다. `dist` 폴더와 exe 파일은 소스 저장소에 포함하지 않고, 배포 담당자가 GitHub Release 첨부 파일로 올립니다.

## 다운로드가 막힐 때

Windows 또는 Microsoft Edge에서 `일반적으로 다운로드되지 않음`, `다운로드할 수 없음`, SmartScreen 경고가 나올 수 있습니다. 현재 실행파일은 새로 빌드한 서명되지 않은 exe라서 Microsoft 평판 정보가 충분하지 않을 수 있습니다.

- 긴 `release-assets.githubusercontent.com/...` 주소는 GitHub가 잠시 발급하는 임시 주소입니다. 공유하거나 Microsoft 피드백에 적을 때는 위의 GitHub Releases 주소를 사용합니다.
- 내려받은 파일명이 `VHLookupLocal_pivot_v1.1.exe`인지 확인합니다.
- PowerShell에서 아래 명령으로 파일 해시를 확인합니다.

```powershell
Get-FileHash .\VHLookupLocal_pivot_v1.1.exe -Algorithm SHA256
```

현재 Release 기준 SHA256은 아래 값입니다.

```text
898191341AAAD542910C78224ABFA6812289919057BC0151C4E64CFF721A4C54
```

공식 Release에서 받은 파일이고 SHA256이 일치하면 Edge 다운로드 목록에서 `유지` 또는 `그래도 유지`를 선택할 수 있습니다. 실행 시 SmartScreen이 뜨면 `추가 정보`를 누른 뒤 실행할 수 있습니다. 기관 PC에서 계속 차단되면 보안 담당자에게 공식 Release 주소와 SHA256을 전달해 허용을 요청해야 합니다.

직접 빌드한 경우에는 아래 파일을 실행합니다.

```text
dist\VHLookupLocal_pivot_v1.1.exe
```

## 사용 순서

1. 전달받은 `VHLookupLocal_pivot_v1.1.exe`를 더블클릭합니다. 직접 빌드한 경우에는 `dist\VHLookupLocal_pivot_v1.1.exe`를 실행합니다.
2. 실제 업무 파일은 아래 버튼 중 하나를 선택합니다.
   - `1. 개인정보 마스킹`
   - `2. 분류별 시트 나누기`
   - `3. 엑셀/CSV 파일 여러 개 합치기`
   - `4. 전/후 파일 검증`
   - `5. 피벗 요약표 만들기`
3. 파일을 올린 뒤 같은 창의 미리보기를 확인합니다.
4. 자동 추천이 틀리면 `컬럼 매칭 수정`을 눌러 드롭박스로 수정합니다.
5. `실행`을 누릅니다.
6. 저장 창에서 결과 파일을 저장할 폴더와 파일명을 선택합니다. 취소하면 결과 파일을 만들지 않습니다.

행/열 합치기에서 어느 한쪽에만 있는 행이나 열은 삭제하지 않습니다. 결과 시트에 그대로 남기고, 빈 칸은 노란색과 메모로 확인 표시합니다.
합치기 결과 첫 시트에는 `원본 파일명`, `원본 시트명`, `원본 행 번호` 같은 내부 확인용 컬럼을 따로 표시하지 않습니다.

## 1번 개인정보 마스킹 사용법

1. `1. 개인정보 마스킹`을 누릅니다.
2. 개인정보를 가릴 엑셀/CSV 파일을 선택합니다.
3. 결과 파일의 `마스킹결과` 시트에서 가려진 값을 확인합니다.
4. 노란색 셀의 메모를 열면 어떤 유형이 마스킹됐는지 볼 수 있습니다.
5. `마스킹내역` 시트에는 원본 값 없이 위치와 유형만 기록됩니다.

주요 표시 예시는 아래와 같습니다.

| 원본 | 결과 |
| --- | --- |
| `홍길동` | `***` |
| `hong@example.go.kr` | `***` |
| `서울특별시 중구 세종대로 110` | `***` |
| `010-1234-5678` | `***` |
| `110123456789` | `***` |
| `900101-1234567` | `******-*******` |
| `생년월일 1900/01/01` | `생년월일 ***` |

## 3번 파일 합치기 사용법

1. `3. 엑셀/CSV 파일 여러 개 합치기`를 누릅니다.
2. `파일 올리기`로 합칠 파일을 선택합니다.
3. 아래 옵션 중 하나를 선택합니다.
   - `자동 선택`: 파일 구조를 보고 행 합치기 또는 열 합치기를 추천합니다.
   - `행 합치기`: 같은 양식의 여러 파일을 아래로 이어 붙입니다.
   - `열 합치기`: 공통 키를 찾아 다른 파일의 열을 오른쪽에 붙입니다.
4. 파일을 올리면 같은 창의 `미리보기` 영역에 예상 결과 10행이 바로 표시됩니다.
5. 자동 매칭이 틀리면 `컬럼 매칭 수정`을 눌러 드롭박스로 수정합니다.
6. `실행`을 누른 뒤 저장 창에서 결과 파일 위치와 이름을 선택합니다.
7. 결과 파일은 `결과` 시트를 먼저 보고, 자동 매칭 근거와 확인할 점은 `확인사항` 시트에서 봅니다.

## 4번 전/후 파일 검증 사용법

1. `4. 전/후 파일 검증`을 누릅니다.
2. `수정 전` 파일과 `수정 후` 파일을 각각 선택합니다.
3. 두 파일을 모두 선택하면 같은 창의 `미리보기` 영역에 차이 10행이 바로 표시됩니다.
4. 자동 매칭이 틀리면 `컬럼 매칭 수정`을 눌러 비교 기준열과 전/후 컬럼 매칭을 수정합니다.
5. `실행`을 누른 뒤 저장 창에서 결과 파일 위치와 이름을 선택합니다.
6. 결과 파일의 `후파일_메모` 시트에서 노란색 셀의 메모를 확인합니다.
7. 값 변경, 행 추가/누락, 컬럼 변경은 `확인사항` 시트에서 확인합니다.

## 5번 피벗 요약표 만들기 사용법

1. `5. 피벗 요약표 만들기`를 누릅니다.
2. `파일 선택`으로 요약할 엑셀/CSV 파일을 선택합니다.
3. 드롭박스에서 `행 기준`, `열 기준`, `값 열`, `집계 방식`을 선택합니다.
4. `건수`는 값 열 없이 행 개수를 셉니다.
5. `합계`, `평균`, `최대`, `최소`는 숫자 값 열을 선택합니다.
6. 같은 창의 `미리보기`에서 요약표를 확인한 뒤 `실행`을 누르고 저장 위치를 선택합니다.
7. 결과 파일은 `피벗요약` 시트를 먼저 보고, 기준 설명과 확인할 점은 `확인사항` 시트에서 봅니다.

## 중요한 점

- 원본 엑셀/CSV 파일은 수정하지 않습니다.
- 파일 합치기는 선택한 파일만 합칩니다. 폴더 안의 다른 파일을 마음대로 끼워 넣지 않습니다.
- 일반 엑셀/CSV 합치기는 원본 열을 삭제하지 않습니다.
- 일반 엑셀/CSV 합치기 결과 첫 시트에는 내부 추적용 기본정보 컬럼을 표시하지 않습니다.
- 키 컬럼과 가져올 컬럼은 기본적으로 자동 추천합니다.
- 실행 전 미리보기 화면에서 자동 추천된 컬럼 매칭을 사람이 수정할 수 있습니다.
- 합치기, 정리, 대조 결과는 첫 시트가 실제 결과입니다.
- 자동 매칭 근거, 확인할 점, 개인정보 의심 요약은 `확인사항` 시트 한 장에 모읍니다.
- 개인정보 마스킹 결과의 `마스킹내역` 시트에도 원본 개인정보 값은 저장하지 않습니다.
- 샘플 데이터는 프로그램 버튼으로 따로 만들지 않고 `samples\public_admin` 폴더에 파일로 제공합니다.
- 샘플 폴더는 실행 화면과 같은 순서로 `01_privacy_masking`부터 `05_pivot_summary`까지 나뉘어 있습니다.
- 피벗 샘플은 `samples\public_admin\05_pivot_summary\budget_execution.csv`입니다.
