# VLOOKUP/HLOOKUP Killer Local 보안 검토 설명서

이 문서는 공공기관, 교육청, 학교, HR, 총무, 예산 부서에서 내부 보안 검토 시 설명할 수 있도록 작성한 요약 자료입니다.

## 운영 원칙

- AI 모델을 호출하지 않습니다.
- 외부 API를 호출하지 않습니다.
- 클라우드 저장소에 업로드하지 않습니다.
- 사용자가 선택한 로컬 파일만 읽습니다.
- 원본 엑셀/CSV 파일은 수정하지 않습니다.
- 결과 파일은 사용자가 지정한 로컬 경로에 새로 저장합니다.

## 데이터 처리 범위

처리 대상:

- `.xlsx`
- `.xlsm`
- `.csv`

처리 방식:

- 엑셀 파일은 값 중심으로 읽습니다.
- `.xlsm` 파일의 매크로를 실행하지 않습니다.
- 제목 행, 안내문 행, 헤더 행을 로컬 메모리에서 탐지합니다.
- 컬럼명 동의어, 문자열 유사도, 데이터 겹침을 기준으로 키 컬럼과 값 컬럼을 추천합니다.
- 병합, 수합, 누락 확인, 가로표 변환을 로컬 프로세스에서 수행합니다.

## 로그와 리포트 원칙

기본 로그와 결과 리포트에는 민감한 셀 값을 상세 정보로 저장하지 않습니다.

기본 리포트에 포함되는 정보:

- 처리 파일 수
- 성공/실패 파일 수
- 결과 행 수
- 확인 필요 건수
- 자동 추천된 키 컬럼명
- 자동 매칭된 컬럼명
- 개인정보로 보이는 컬럼/패턴의 유형과 감지 건수
- 파일명, 시트명, 원본 행 번호
- 오류 유형과 조치 안내

기본 리포트에 저장하지 않는 정보:

- 주민등록번호/외국인등록번호
- 성별
- 나이
- 생년월일
- 계좌번호
- 개인 휴대폰 번호의 상세값
- 민감한 셀 값의 원문 상세
- 외부 전송 로그
- 개인정보 원문 값

## 개인정보 점검

결과 엑셀에는 필요한 경우 `확인사항` 시트에 개인정보 의심 요약을 표시합니다.

점검 대상:

- 이름/담당자/직원명 컬럼
- 연락처/전화번호/휴대폰 컬럼
- 이메일 컬럼
- 주소 컬럼
- 계좌/은행/예금주 컬럼
- 주민등록번호/외국인등록번호로 보이는 패턴
- 생년월일로 보이는 문맥 패턴

점검 결과에는 컬럼명, 점검 유형, 감지 건수, 조치 안내만 남깁니다. 셀의 실제 개인정보 값은 저장하지 않습니다.

## 자동 추천 검토 가능성

자동 추천은 블랙박스로 처리하지 않습니다. 결과 엑셀의 `확인사항` 시트에 다음 정보를 남깁니다.

- 어떤 컬럼을 키 컬럼으로 추천했는지
- 기준표 컬럼과 대상표 컬럼을 어떻게 연결했는지
- 추천 신뢰도
- 추천 방식
- 사용자가 검토해야 할 메모

## 원본 보호

원본 파일은 읽기 전용 입력으로 취급합니다.

- 원본 파일에 수식을 삽입하지 않습니다.
- 원본 파일의 시트, 셀, 서식을 수정하지 않습니다.
- 결과는 새 `.xlsx` 파일로 저장합니다.
- 오류가 발생한 파일이 있어도 전체 작업을 중단하지 않고 `확인사항` 시트에 기록합니다.

## 제한 사항

- 암호화된 파일은 처리하지 못할 수 있습니다.
- 손상된 파일은 `파일 처리 실패`로 기록됩니다.
- 자동 추천 신뢰도가 낮은 경우 사용자가 미리보기와 `확인사항` 시트를 확인해야 합니다.
- 외부 시스템 업로드, 크롤링, 기관별 업무 시스템 직접 입력은 제공하지 않습니다.

## 보안 검토용 한 문장

이 도구는 AI, API, 클라우드 없이 사용자가 선택한 로컬 엑셀/CSV 파일만 읽고, 원본을 수정하지 않은 채 결과 엑셀과 확인 리포트를 사용자의 PC에 생성하는 로컬 업무 자동화 프로그램입니다.
