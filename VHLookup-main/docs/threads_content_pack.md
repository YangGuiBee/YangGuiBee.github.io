## Short Posts

### 1. VLOOKUP 피로

행정직 엑셀 업무에서 진짜 힘든 건 어려운 함수가 아니라 매달 반복되는 대조입니다.

기준명단과 제출명단을 고르면 `사번`과 `직원번호`처럼 이름이 달라도 자동으로 맞추고, 부서/직급/소속을 붙여줍니다.

틀릴 수 있는 건 첫 시트에 색과 메모로 표시하고, 왜 그렇게 맞췄는지는 `확인사항`에 남깁니다.

### 2. 보안 포인트

공공기관용 엑셀 자동화에서 중요한 건 기능보다 설명 가능성입니다.

이 도구는:

- AI 호출 없음
- 외부 API 없음
- 클라우드 업로드 없음
- 원본 파일 수정 없음
- 결과는 로컬 xlsx로 저장

이 원칙으로 만들고 있습니다.

### 3. 오류 리포트

엑셀에서 `#N/A`를 보는 순간 실무자는 다시 수작업을 시작합니다.

그래서 오류를 이렇게 바꿨습니다.

- 기준표에 없음
- 필수값 누락
- 숫자 오류
- 날짜 오류
- 중복 제출 의심
- 형식 불일치

결과 파일의 `확인사항` 시트만 보면 됩니다.

### 4. 개인정보 점검

공공기관 엑셀 자동화에서 민감한 건 “처리했다”보다 “어떤 개인정보가 들어 있는지 확인할 수 있느냐”입니다.

그래서 결과 파일의 `확인사항` 시트에 개인정보 의심 요약을 넣었습니다.

실제 값은 복사하지 않고:

- 이름 컬럼
- 연락처 컬럼
- 이메일 컬럼
- 계좌 관련 컬럼
- 고유식별번호 패턴
- 성별/나이/생년월일 컬럼
- 생년월일 문맥 패턴

처럼 유형과 건수만 표시합니다.

### 5. 제출자료 수합

제출자료 수합이 어려운 이유:

- 어떤 파일은 1행이 제목
- 어떤 파일은 2행이 안내문
- 컬럼명이 `기관명`, `제출기관`, `학교명`으로 제각각
- 금액에 문자 들어감
- 날짜 형식 제각각

이런 케이스를 샘플로 넣고, 한 번에 결과 엑셀 8개가 생성되게 만들었습니다.

### 6. 샘플 유도

사용자는 설명보다 샘플을 먼저 봐야 합니다.

그래서 더블클릭용 `run_demo.bat`을 넣었습니다.

실행하면 `demo_output`에:

- 제출자료 수합
- 오류 검증
- 교육이수 대조
- 누락자 확인
- 수당/예산 대조
- 월별 가로표 변환

결과 엑셀이 바로 생깁니다.

## Replies To Expected Questions

### "AI 쓰나요?"

아니요. AI 모델을 호출하지 않습니다. 컬럼명 동의어, 문자열 유사도, 데이터 겹침을 로컬에서 계산합니다.

### "파일이 밖으로 나가나요?"

아니요. 사용자가 선택한 로컬 파일만 읽고, 결과도 로컬 폴더에 새 xlsx로 저장합니다.

### "원본 엑셀에 수식 넣나요?"

아니요. 원본 파일은 수정하지 않습니다. 결과 파일을 따로 만듭니다.

### "컬럼명을 매번 지정해야 하나요?"

기본은 자동 추천입니다. `사번`과 `직원번호`, `기관명`과 `제출기관`처럼 흔한 표현은 자동으로 맞춥니다. 낮은 신뢰도와 추천 근거는 `확인사항`에서 확인할 수 있게 남깁니다.

### "실무 샘플 있나요?"

있습니다. `run_demo.bat`을 실행하면 공공기관 행정직 업무 기준 샘플 결과 엑셀 8개가 생성됩니다.
