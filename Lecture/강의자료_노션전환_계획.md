# 강의자료 노션(Notion) 전환 — 최종 계획서

> 조건: Windows / 작성·강의연출 둘 다 중요 / 대면·온라인 혼합 / 학생은 이메일 인증코드 로그인, 관리자 권한 분리 / 비용 0원 목표 / GitHub 병행
>
> 대상 교안: [ML](https://github.com/YangGuiBee/ML) (16주차·100% Jupyter Notebook), [AIG](https://github.com/YangGuiBee/AIG) (16주차·법/국제표준 텍스트 중심)

---

## 1. 최종 아키텍처 (3계층 + 단일 원본)

| 계층 | 역할 | 도구 |
|---|---|---|
| **작성·표현** | 교안을 편하고 예쁘게 작성, **모든 원본 보관** | **Notion** (단일 원본 / source of truth) |
| **공개 배포** | 로그인 없이 누구나 보는 **공개용 PDF**만 게시 | **GitHub** |
| **강의 실연** | 화면에 노션 띄우고 줌·화면 메모 | **노션 화면 + ZoomIt** |

> 핵심 원칙: **원본은 노션 하나.** GitHub에는 노션에서 파생된 산출물만 **단방향**으로 올린다 → 데이터 중복·불일치 없음(무결성 확보).

## 2. 접근 등급 & 채널 (가장 중요한 설계)

자료를 "전부 보호"하려 하지 말고 **민감도 등급**으로 나눠 채널을 배정한다.

| 등급 | 예시 | 채널 | 접근 |
|---|---|---|---|
| **공개 가능** | 일반 강의노트, 참고자료, 예제코드 | **GitHub**(PDF), 또는 노션 공개 게시 | 로그인 불필요 |
| **수강생 한정** | 배포 교안, 과제 안내 | **노션 게스트 로그인**(이메일코드) | 초대된 학생만 |
| **민감/통제** | 시험·정답, 저작권 자료 | **eCampus** (또는 노션 게스트) | 기관 통제 |

> ⚠️ **가드레일**: GitHub 공개 커밋 = 사실상 **영구·전 세계 공개**(검색 색인·복제·히스토리 잔존, 삭제해도 완전 제거 안 됨). 저작권·시험·정답은 **절대 GitHub 금지**.

> 현실적 기대치: 강의자료는 완전 보안이 불가능(로그인해도 캡처·다운로드 가능). 목표는 **접근 통제**이지 복제 방지(DRM)가 아니다.

## 3. 원하던 기능별 구현 방법

**작성 표현 (노션)**
- 글자 색: 텍스트/배경 팔레트(약 10색). 강조는 콜아웃 블록 활용. (자유 HEX는 불가)
- 볼드·밑줄: `Ctrl+B` / `Ctrl+U` / 기울임 `Ctrl+I` / 취소선 `Ctrl+Shift+S`
- 이미지 크기: 모서리 드래그로 자유 조정
- 새창 링크: 웹 공개 페이지에서 외부 링크 기본 새 탭

**강의 실연 (ZoomIt, Windows 무료)**
- Prezi식 줌: ZoomIt 화면 확대로 "축소→확대" 효과 대체 (특정 강의만 Prezi 임베드도 가능)
- 마우스 밑줄·강조: ZoomIt 그리기, **ESC로 즉시 삭제**
- 노션은 사이드바 접고 **브라우저 전체화면**으로 띄우면 발표용으로 깔끔

## 3-A. AI 보조 작성 워크플로우 (NotebookLM · Claude Project)

교안 작성 단계에서 AI를 활용해 초안을 만들고 노션에 적재한다. 두 도구는 단계가 다르므로 **함께** 쓴다.

| 단계 | 도구 | 역할 |
|---|---|---|
| **1. 근거 수집·정리** | **NotebookLM** | 참고자료(AIG 법령·국제표준 등) 업로드 → **인용 포함** 요약·학습가이드. 사실 정확성 강점 |
| **2. 집필·노션 저장** | **Claude Project** | 정리본 + 강의 수준·문체 지시로 교안 초안 작성 → **Notion 커넥터로 노션에 직접 저장** |

- "작성 후 노션 저장"만 보면 **Claude Project가 우위** — NotebookLM은 네이티브 노션 저장이 없고(구글 독스만), 노션은 확장프로그램/복붙이 필요. Claude는 커넥터(OAuth)로 노션 페이지를 바로 생성.
- ML(코드·예제 비중↑)은 Claude 주도, AIG(법·표준)는 NotebookLM의 출처 근거가 가치.
- ⚠️ AIG는 법·표준이라 **최종본은 1차 출처로 검증** 필수.

## 4. 무료 운영 — 노션 교육용 Plus

- 인증 대학의 **교수/교사는 Plus 플랜 무료** → **게스트 최대 100명** (수강생 12~21명 충분 커버), 비용 0원.
- 자격: **ac.kr 교육기관 이메일** + 학교가 WHED 등재(국내 대학 대부분). 다른 증빙 없이 이메일로만 인증.
- 구조: 워크스페이스는 **본인 1인 멤버(소유자=관리자)**, 학생은 **읽기 전용 게스트**로 초대 → 이메일 코드로 로그인 조회. (게스트는 멤버 수에 미포함)

## 5. 노션 → GitHub 백업 방식

노션을 **Markdown으로 내보내면 색·밑줄·콜아웃·이미지크기 등 시각효과는 손실**(GitHub이 inline 색상 제거). 따라서:

| 백업 형식 | 용도 | 비고 |
|---|---|---|
| **PDF** | 공개 배포 + 만든 모습 그대로의 시각 스냅샷 | GitHub 게시 기본 |
| **Markdown** (선택) | 텍스트·구조의 버전관리·검색·diff | 효과는 빠지지만 내용 보존 |

> 정석: **원본=노션, GitHub엔 공개 가능한 PDF 게시.**

## 7. 지속성 · 출구 전략

- **콘텐츠는 사라지지 않는다.** 노션 워크스페이스는 계정 귀속 → 계약 종료 전 **계정 이메일을 개인 이메일로 변경**하면 소유권·자료 유지. 잃는 건 데이터가 아니라 *교육용 Plus 혜택*(게스트 100명)뿐 → 무료 일반 플랜으로 다운그레이드(삭제 아님).
- **학생 접근**은 학기제 정상 동작. 매 학기 그 학기 수강생 이메일로 새로 초대, 졸업생은 자연 종료. 특정 졸업생 유지 필요 시 개인 이메일로 초대.
- **영구 안전장치**: GitHub(공개 PDF) + 정기 내보내기 백업. 노션 떠나도 자료가 갇히지 않음.
